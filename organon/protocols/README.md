# Protocols

> Step-by-step procedures for common documentation tasks.

Protocols are the **"how"** layer of the organon — actionable steps that LLMs and developers can follow to accomplish specific tasks.

---

## Available Protocols

| Protocol | When to Use |
|----------|-------------|
| [semantic-mapping.md](./semantic-mapping.md) | Writing semantic mapping tables in ETHOS files |
| [add-component-docs.md](./add-component-docs.md) | Creating documentation for a new component |
| [regenerate-docs.md](./regenerate-docs.md) | Regenerating docs and fixing CI failures |

---

## Protocol Structure

Each protocol follows this format:

1. **When to Use** — Trigger conditions
2. **Procedure** — Numbered steps
3. **Verification** — How to confirm success
4. **Troubleshooting** — Common issues and fixes
5. **See Also** — Related protocols and references
